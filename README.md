Лабораторная работа №7–8

Управление связанным списком блоков управления процессами (PCB)

Цель работы:
Целью лабораторной работы является разработка консольного приложения на языке C++, реализующего управление связанным списком блоков управления процессами (PCB) с использованием принципов объектно-ориентированного программирования, указателей и динамического выделения памяти.

В рамках работы необходимо реализовать:
 
 - структуру PCB (Process Control Block) для хранения информации о процессе;

 - связанный список узлов, каждый из которых содержит объект PCB;

 - класс для управления списком процессов с возможностью вставки, удаления и вывода данных.

Структура PCB содержит следующую информацию о процессе:

 - processID — уникальный идентификатор процесса (int);

 - processName — название процесса (std::string);

 - processStatus — состояние процесса (std::string);

 - commandCounter — счётчик команд (int);

 - cpuRegisters — массив значений регистров CPU (int[4]).

Класс ListNode представляет узел связанного списка и содержит:

 - data — объект типа PCB, хранящий данные процесса;
 
 - next — указатель на следующий узел списка (ListNode*).

Также реализован конструктор для инициализации узла и деструктор.

Класс ProcessList предназначен для управления связанным списком процессов.

Методы:

 - insert(const PCB& newPCB)
Вставляет новый процесс в список, упорядочивая элементы по processID.
Возвращает true, если вставка успешна, и false при попытке вставки дубликата.

 - remove(int pid)
Удаляет процесс по идентификатору.
Возвращает true, если процесс найден и удалён, и false в противном случае.

 - printList()
Выводит информацию обо всех процессах, находящихся в списке.

В функции main() выполняются следующие действия:

 1. Создание объекта списка процессов ProcessList;
 2. Добавление нескольких процессов в список;
 3. Вывод содержимого списка;
 4. Удаление одного из процессов;
 5. Повторный вывод списка;
 6. Демонстрация неудачных операций (удаление несуществующего элемента, попытка вставки дубликата);
 7. Финальный вывод состояния списка.

Таким образом демонстрируется корректность работы всех реализованных методов.
